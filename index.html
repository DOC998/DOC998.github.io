BFS + 动态规划（？）

赛场改了半天记忆化搜索，最后十分钟改了个bfs莫名其妙过了就离谱….

soulution:

起点ax,ay 终点px,py

$dp[x][y]$表示从该点到终点的最少修改次数，

初始化$dp[px][py]=0$,其余为0x3f3f3f3f（无穷大）,

然后层序遍历周围的点(bfs),

如果方向一致就修改为,$dp[px][py]$

否则为$dp[px][py]+1$，然后加入队列

才怪才怪才怪才怪才怪

因为这个点也许已经被修改过，

所以需要判断这个点原本的答案是否更优，

如果更优就修改它，然后重新加入队列，

这样大概可以保证所有点都是最优的,

最后输出$dp[ax][ay]$即可

prime keys :和传统bfs不同，我们没有采用st标记，

是否可以重复遍历某个点，是根据$dp[x][y]$的值判断的，

最好情况$O(n)$ 最坏自己算 蒟蒻不会算

WDNMD算法之动态规划

1.根据数据范围猜循环和数组

原则：别超过1e6就彳亍

2e5~1e6：一维

1e3：二维

1e2：三维

2.内外循环顺序要点

新状态不能影响用于递推当前层内新状态的旧状态

3.大到小or小到大？

取决于递推关系。

如：

官方题解定义dp状态如下 ：

\(dp[ f ] [ i ]\):已经刚好花费了f，到达地点 i 的所有方案数（由于dp属性为sum而非min/max，故无需考虑初始化问题）

初始化：\(d[0][start]=1\)

状态转移：\(dp[f][i]+=dp[f-dis][j]\) dis为j到i的花费

显然，由于大的状态需要用小的来递推（对fuel），故采用小到大递推

鉴于【已经消费】不是很容易想到（菜就是菜），我们也可以采用更容易理解的大到小递推：

状态定义：

\(dp[i][f]\)到达i时正好剩下f的方案数

状态转移：

\(dp[i][f]=dp[j][f+dis]+dp[i][f]\)

显然，由于大的状态需要用小的来递推（对fuel），故采用大到小递推：

要点：

题意转化为dp题

数组内i j k顺序和对答案无影响，取决于循环

dp实际上最重要的是最外层的循环，比如各种背包最外层是当前物品，然后枚举空间情况；本题看似是对n个地点讨论，实际上油的数量才是关键：因为各个地点是相互连通的，更新和之前所有节点有关，情况复杂；而对油枚举则类似于背包最外层，当前状态仅取决于之前的某层。所以最外层需要对油的数量进行枚举，即油才是那个“物品”，而n个地点是“空间”。

最外层的枚举顺序由递推关系表达式确定：只能由已知推未知。

状态定义和递推表达式靠guess和imagine

Posted on 2022-03-10

题目：https://leetcode-cn.com/problems/solving-questions-with-brainpower

给你一个下标从 0 开始的二维整数数组 questions ，其中 questions[i] = [pointsi, brainpoweri] 。

这个数组表示一场考试里的一系列题目，你需要 按顺序 （也就是从问题 0 开始依次解决），针对每个问题选择 解决 或者 跳过 操作。解决问题 i 将让你 获得 pointsi 的分数，但是你将 无法 解决接下来的 brainpoweri 个问题（即只能跳过接下来的 brainpoweri 个问题）。如果你跳过问题 i ，你可以对下一个问题决定使用哪种操作。

比方说，给你 questions = [[3, 2], [4, 3], [4, 4], [2, 5]] ： 如果问题 0 被解决了， 那么你可以获得 3 分，但你不能解决问题 1 和 2 。 如果你跳过问题 0 ，且解决问题 1 ，你将获得 4 分但是不能解决问题 2 和 3 。 请你返回这场考试里你能获得的 最高 分数。
